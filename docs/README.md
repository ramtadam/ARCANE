# Dokumentacja

Ten katalog jest głównym miejscem wiedzy o projekcie ARCANE.

## Zakres
- architektura wysokopoziomowa,
- decyzje architektoniczne,
- analizy narzędzi i repozytoriów,
- specyfikacje interakcji agentów,
- propozycje implementacji,
- materiały edukacyjne,
- roadmapa i planów rozwoju.

## Struktura dokumentacji
- architecture/ — opis architektury systemu i jego warstw,
- ADR/ — decyzje architektoniczne i uzasadnienia,
- TAR/ — analizy inspiracji z zewnętrznych repozytoriów,
- AIS/ — specyfikacje komunikacji agentów,
- IC/ — kandydaci do implementacji,
- learning/ — prywatny podręcznik i notatki rozwojowe,
- sdk/ — koncepcje SDK i interfejsów,
- plugins/ — koncepcje rozszerzeń i integracji,
- testing/ — strategia testów i walidacji,
- deployment/ — plany wdrożeń i środowisk,
- roadmap/ — kamienie milowe projektu,
- developer-guide/ — instrukcje dla deweloperów.

## Zasada organizacji
Każdy podkatalog odpowiada innej klasie wiedzy. Dokumentacja jest traktowana jako pierwszy artefakt projektu, a nie dodatki do kodu.
